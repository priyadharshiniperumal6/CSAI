# LLM UI Generation Guide (Universal Theme DS)

This section provides **LLM-facing context** for view generation. The goal is to make component selection
**deterministic**, consistent with the design system, and aligned with UX best practices.

## How To Use

Provide these documents as context before generating UI code or layouts. They guide
component selection and layout decisions at the **view level** (screens, pages, dashboards).

## Primary Rules

1. Prefer **existing Universal Theme components** over custom HTML.
2. Treat **`HostShell` as the default top-level application layout** for app/page/route generation.
3. Choose the **lowest-level component** that matches the task, then compose.
4. Use **`UniEChart`** for analytical charting and **`UniReactFlow`** for flow-building experiences.
5. Follow **data-display heuristics**: dense/comparative data → tables; summary/variable content → cards.
6. If a component exists in the library, **use it as-is** and do not re-style it unless instructed.
7. If unsure, **ask for missing context** (data shape, user task, density, target device).

## Required Inputs For Good Generation

Provide as much of the following as possible:
- Primary user goal (scan, compare, edit, explore, create)
- Data shape (rows, columns, nested objects, media)
- Density requirements (high, medium, low)
- Interactions required (filter, sort, select, edit, bulk actions)
- Device constraints (desktop, tablet, mobile)
