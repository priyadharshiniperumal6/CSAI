# Figma Tokens Syncronization process

> **Current state (code-first tokens):** Token updates are authored in `packages/tokens/src/uni-token.js`, then runtime artifacts are generated via `pnpm tokens:generate`.  
> This page documents the older Figma-first GitHub Action flow for historical context.

The synchronization are divided to two parts:
  - running the github action
  - review the PR that will appear after the github action runs


### GitHub Action Documentation: Sync Figma Variables

#### Overview

This GitHub Action automates the synchronization of variables from a Figma file to a repository. It triggers on manual invocation (`workflow_dispatch`), allowing the user to input a file name, collection name, mode, GitHub user credentials, and more. The workflow involves setting up a Node environment, installing dependencies, syncing data from Figma, and creating a pull request with the changes.

#### Workflow Breakdown:

1. **Trigger**:
   - The action is triggered manually via `workflow_dispatch`. Inputs are provided by the user, including the Figma file name (`fileName`), collection name (`collectionName`), mode, and GitHub credentials (user name and email).

2. **Job: Deploy**
   - **Runs on**: `ubuntu-latest`.

   **Steps:**
   1. **Setup Node.js Environment**:
      - Uses the `setup-node@v3` action to install Node.js version 18.
      <br/>
   2. **Checkout Repository**:
      - Uses `checkout@v3` to check out the repository.
      <br/>
   3. **Install pnpm**:
      - Sets up the `pnpm` package manager (version 8.15.4), using the specified `package.json`.
      <br/>
   4. **Install Dependencies**:
      - Runs `pnpm install` to install necessary dependencies from `package.json`.
      <br/>
   5. **Set Environment Variable**:
      - Exports the Figma file name (`fileName`) into the GitHub Actions environment.
      <br/>
   6. **Sync with Figma**:
      - The `sync-figma` script is executed, using the Figma API token (`FIGMA_TOKEN`), collection name (`collectionName`), and mode provided by the user. This step handles the actual synchronization of variables from the Figma document.
      <br/>
   7. **Create New Branch**:
      - A new branch is created, using the current date and time as part of the branch name (`figma-variables-YYYYMMDDHHMMSS`), and the branch name is stored in the environment.
      <br/>
   8. **Commit Changes**:
      - Changes are staged and committed using the provided GitHub user name and email. The changes are pushed to the newly created branch.
      <br/>
   9. **Create Pull Request**:
      - A pull request (PR) is automatically created against the `main` branch. The PR includes a title and body indicating it was generated by this action.
      <br/>
#### Key Inputs:
- **fileName**: The Figma file key used to fetch variables.
- **collectionName**: Specifies which variable collection from Figma to sync.
- **mode**: Optional input for specifying a collection mode (defaults to `Mode 1`).
- **gitUserName**: GitHub username for committing changes.
- **gitUserEmail**: Email associated with the GitHub account, used for commits.

#### Key Environment Variables:
- **FIGMA_TOKEN**: Secret token to authenticate with Figma's API.
- **GITHUB_ENV**: Used to store the branch name for later steps.

#### Key Commands:
- **pnpm install**: Installs project dependencies.
- **pnpm run sync-figma**: Executes the script responsible for syncing Figma variables.

#### Outputs:
- A new branch with synchronized Figma variables.
- A pull request that contains the changes, ready for review.

This workflow provides an automated solution for syncing design variables from Figma to a code repository, minimizing manual updates and ensuring consistency between design and code.
